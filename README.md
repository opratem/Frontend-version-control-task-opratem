# Frontend Version Control Task

This repository is for practicing Git version control workflows, branch management, pull requests, and collaboration using GitHub. It demonstrates feature branch creation, commits, PR review, merging, reverting commits, and branch renaming.

## Branches

- `master` — Main branch containing the stable, merged project files.
- `feature-header` — Implemented the header section (merged into master).
- `feature-footer-updated` — Implemented the footer section (renamed from feature-footer).

## Common Git Commands Used

- `git clone <repo_url>` — Clone the repository
- `git checkout -b <branch>` — Create and switch to a new branch
- `git add <file>` — Stage changes
- `git commit -m "message"` — Commit changes
- `git push -u origin <branch>` — Push branch to remote
- `git pull` — Fetch and merge updates
- `git revert <commit>` — Revert a specific commit
- `git branch -m <old> <new>` — Rename branch locally
- `git push origin :<old>` — Delete old branch remotely
